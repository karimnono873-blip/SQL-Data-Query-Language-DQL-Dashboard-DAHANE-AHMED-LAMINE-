The objective of this checkpoint is to practice SQL Data Query Language (DQL) by writing queries to retrieve and analyze data from the previously created employee participation information system database. These queries will help you gain insights into employee, department, and project relationships, and summarize data for decision-making purposes.

Query 1: Employees on Multiple Projects

Instruction: Write a query to retrieve the names of employees who are assigned to more than one project, including the total number of projects for each employee.

Logic: We JOIN the Employee table to the associative Employee_Project table. We then GROUP BY the employee and use the HAVING clause to filter the aggregated COUNT to strictly greater than 1.

📄 01_multi_projects.sql
SELECT E.Name, COUNT(EP.Project_Num_P) AS Total_Projects FROM Employee E JOIN Employee_Project EP ON E.Num_E = EP.Employee_Num_E GROUP BY E.Num_E, E.Name HAVING COUNT(EP.Project_Num_P) > 1;

Query 2: Projects by Department

Instruction: Write a query to retrieve the list of projects managed by each department, including the department label and manager’s name.

Logic: Since projects are directly assigned to departments via the Department_Num_S foreign key, this only requires a standard INNER JOIN between the two parent/child tables.

📄 02_dept_projects.sql
SELECT D.Label AS Department_Name, D.Manager_Name, P.Title AS Project_Title FROM Department D JOIN Project P ON D.Num_S = P.Department_Num_S;

Query 3: "Website Redesign" Team

Instruction: Write a query to retrieve the names of employees working on the project "Website Redesign," including their roles in the project.

Logic: This requires chaining two joins to traverse the Many-to-Many relationship. We start at Employee, pass through the Employee_Project bridging table to get the Role, and end at Project to filter by the specific string title.

📄 03_project_team.sql
SELECT E.Name, EP.Role FROM Employee E JOIN Employee_Project EP ON E.Num_E = EP.Employee_Num_E JOIN Project P ON EP.Project_Num_P = P.Num_P WHERE P.Title = 'Website Redesign';

Query 4: Largest Department

Instruction: Write a query to retrieve the department with the highest number of employees, including the department label, manager name, and the total number of employees.

Logic: We join Departments and Employees, group by the Department details, calculate the total count, order the results in descending order (highest first), and limit the output to the single top row.

📄 04_largest_dept.sql
-- Note: LIMIT 1 is used in MySQL/PostgreSQL. For SQL Server, use SELECT TOP 1. SELECT D.Label, D.Manager_Name, COUNT(E.Num_E) AS Total_Employees FROM Department D JOIN Employee E ON D.Num_S = E.Department_Num_S GROUP BY D.Num_S, D.Label, D.Manager_Name ORDER BY Total_Employees DESC LIMIT 1;

Query 5: High Earning Employees

Instruction: Write a query to retrieve the names and positions of employees earning a salary greater than 60,000, including their department names.

Logic: A straightforward join to pull the department name, combined with a WHERE clause acting as a numerical threshold filter on the salary column. Notice it is strictly greater than (>), not greater than or equal to.

📄 05_high_earners.sql
SELECT E.Name, E.Position, E.Salary, D.Label AS Department_Name FROM Employee E JOIN Department D ON E.Department_Num_S = D.Num_S WHERE E.Salary > 60000.00;

Query 6: Headcount per Project

Instruction: Write a query to retrieve the number of employees assigned to each project, including the project title.

Logic: We group by the Project data. Best practice dictates using a LEFT JOIN from Project to Employee_Project, ensuring that if a project has zero employees assigned, it still appears in the final report with a count of 0.

📄 06_project_headcounts.sql
SELECT P.Title, COUNT(EP.Employee_Num_E) AS Employee_Count FROM Project P LEFT JOIN Employee_Project EP ON P.Num_P = EP.Project_Num_P GROUP BY P.Num_P, P.Title;

Query 7: Master Role Summary

Instruction: Write a query to retrieve a summary of roles employees have across different projects, including the employee name, project title, and role.

Logic: Similar to Query 3, this requires chaining joins across the M:N relationship. We add an ORDER BY to present the report cleanly, sorting alphabetically by project, and then alphabetically by employee name within that project.

📄 07_role_summary.sql
SELECT E.Name AS Employee_Name, P.Title AS Project_Title, EP.Role FROM Employee E JOIN Employee_Project EP ON E.Num_E = EP.Employee_Num_E JOIN Project P ON EP.Project_Num_P = P.Num_P ORDER BY P.Title, E.Name;

Query 8: Financial Expenditure

Instruction: Write a query to retrieve the total salary expenditure for each department, including the department label and manager name.

Logic: We group by the department hierarchy and use the aggregate math function SUM() on the salary column of the joined employee table to calculate the total departmental overhead.

📄 08_dept_expenditure.sql
SELECT D.Label, D.Manager_Name, SUM(E.Salary) AS Total_Salary_Expense FROM Department D JOIN Employee E ON D.Num_S = E.Department_Num_S GROUP BY D.Num_S, D.Label, D.Manager_Name;
